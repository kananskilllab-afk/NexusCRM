Schema changes & rationale

New tables

quotes — Pre-booking proposal stage. Agents build quotes that can be accepted (→ booking), expired, or rejected. Enables pipeline analytics on conversion rate from proposal to booking, and stores the exact price offered before it becomes a booking.
contact_relationships — Models companion/family/corporate account links between contacts. Essential for group travel, spousal accounts, and B2B clients where one contact is the decision-maker for many travellers.
payment_schedules — Explicit deposit and balance-due installment rows per booking. Previously, financial_events tracked ad hoc payments but couldn't enforce pre-defined payment plan structures (e.g. 20% deposit + 80% 60 days before travel).
supplier_contracts — Per-tenant rate agreements with suppliers: net rates, markup floors, validity windows, and override commission percentages. Decouples the negotiated rate from the supplier's default commission_pct, allowing multiple contracts per supplier per year.
email_templates — Reusable tenant-scoped message templates for confirmations, payment reminders, itinerary shares, and marketing. Supports liquid-style merge variables stored in a jsonb variables column.
email_sends — Delivery and engagement log: sent, delivered, opened, clicked, bounced. Links back to the template, contact, booking, and the triggering user. Required for compliance audit trails and deliverability monitoring.
tag_definitions — Proper tag registry replacing the flat text[] on contacts. Tags have a color, a category (e.g. "Interest", "Lifecycle", "Source"), and belong to a tenant. Enables consistent taxonomy management and UI autocomplete.
contact_tags — Join table between contacts and tag_definitions. Replaces the denormalized text_arr tags column and allows efficient filtering across large contact lists.

Enhancements to existing tables

contacts — Added account_id (self-referencing FK for corporate hierarchy), source (lead origin: referral, web, campaign), lifecycle_stage (enum: lead → prospect → active → lapsed), and dropped the denormalized tags text_arr column in favour of contact_tags.
bookings — Added quote_id FK (traceability from proposal to confirmed booking), group_booking_id (self-referencing FK to link travellers in the same group trip), and cancellation_reason text (captured when status → cancelled for reporting).
booking_segments — Added contract_id FK (links to the supplier_contracts row used to price this segment), cancellation_policy text, and is_on_request boolean (pending supplier confirmation).
users — Added commission_tier (enum: junior/standard/senior/director, drives commission_pct overrides) and phone varchar for 2FA and contact purposes.
suppliers — Added primary_contact_name, primary_contact_phone, and bank_details_encrypted text (AES-256 blob for payment setup, never exposed via API directly).
financial_events — Added payment_schedule_id FK to link ad hoc payments back to an installment in payment_schedules, enabling over/underpayment tracking per installment.
